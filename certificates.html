User Role and Characteristics

Admin

Role:

Admin responsible for managing posts and viewing student interactions.

Characteristics:

  • Has access to upload and manage media posts.
  • Can view and track which students liked their posts.
  • Can perform CRUD operations on posts.
  • Has a unique login credential with higher privileges compared to regular users.

User (Student)

Role:

Students who can view and interact with the content uploaded by admins.

Characteristics:

  • Can view posts uploaded by admins.
  • Can like posts to show engagement.
  • Has a unique login credential but with limited access (view and like features).

Implementation and Deployment

SHC Social Media App

Roles

1. Admin Role

  • The admin has the ability to upload posts that include an image and a description.
  • Admin workflow:
    1. Choose Image: Admin selects an image from their device.
    2. Upload Post UI: The app uploads the selected image to Firebase Storage.
    3. Retrieve Image URL: Once the image is uploaded, Firebase Storage returns the image's URL.
    4. Generate Post Entry: A post entry is created using the image URL, a description, and a timestamp.
    5. Save Post: The post data (image URL, description, and timestamp) is then saved to Firebase Realtime Database.

2. Student/User Role

  • Students/users can only view the posts uploaded by the admin.
  • Student workflow:
    1. View Posts UI: The app retrieves the posts (image URL, description, and timestamp) from Firebase Realtime Database.
    2. Display Posts: The app displays the posts with their associated images and descriptions for students to view.

Firebase Components

  • Firebase Storage:
    • This service stores the images uploaded by the admin.
    • When an image is uploaded, Firebase Storage provides a URL for accessing the image.
  • Firebase Realtime Database:
    • This service stores the post data, including the image URL, description, and timestamp.
    • It allows the app to retrieve posts in real-time and display them in the View Posts UI.
